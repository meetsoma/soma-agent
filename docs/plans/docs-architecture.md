# Docs Architecture — Agent vs Website

## Problem

Docs exist in two places with identical content:
- `agent/docs/` — source, synced to website via `sync-docs.sh`
- `website/src/content/docs/` — generated copies with Astro frontmatter

But they serve different audiences and should be optimized differently.

## Design

### Agent Docs (`agent/docs/`)
**Audience:** The agent itself + users reading docs locally after `npm install`
**Ships with:** `meetsoma` npm package (via `"files": ["docs"]`)
**Optimized for:**
- TL;DR blocks at the top of every doc (agent reads these first)
- Concise — no SEO padding, no narrative fluff
- Quick reference tables over prose
- Internal cross-refs use relative paths (`./protocols.md`)
- Plans stay in `docs/plans/` — internal, never synced

### Website Docs (`website/src/content/docs/`)
**Audience:** Humans browsing soma.gravicity.ai
**Generated by:** `sync-docs.sh` from agent docs
**Optimized for:**
- Narrative flow, examples, guided walkthrough
- SEO: unique descriptions, JSON-LD, internal links
- Cross-refs use website paths (`/docs/protocols`)
- Breadcrumbs, search, sidebar nav
- Changelog synced from repo root

## The Sync

`sync-docs.sh` remains the bridge. It:
1. Reads agent docs (source of truth)
2. Strips the H1 and any existing frontmatter
3. Adds Astro content collection frontmatter (title, description, section, order)
4. Writes to `website/src/content/docs/`

The sync is **one-directional**: agent → website. Never edit website docs directly.

## What Changes

1. **Every agent doc gets a TL;DR block at the top** — between `<!-- tldr -->` markers
   - Agent boot can extract these for quick context injection
   - Website can optionally render them as a summary callout
   - Keeps docs dual-purpose without forking content

2. **Agent docs ship with npm** — add `docs/` to cli package
   - Users can read docs offline: `cat node_modules/meetsoma/docs/protocols.md`
   - Agent can reference them during sessions

3. **Internal plans stay internal** — `docs/plans/` never syncs, never ships

## TL;DR Format

```markdown
# Protocols

<!-- tldr -->
Behavioral rules in `.soma/protocols/`. Loaded by heat: hot=full, warm=breadcrumb,
cold=dormant. Write your own with frontmatter (name, heat-default, applies-to).
Configure thresholds in settings.json.
<!-- /tldr -->

## Full content below...
```
